3. 關鍵互動的動畫實作

---

## App 概述

這是一款 iOS 原生小說閱讀器，核心功能是讀取 .txt 文字檔，提供沉浸式閱讀體驗，並透過繁體中文 TTS 語音在前景與背景朗讀小說。目標使用者是台灣的網路小說重度讀者，單本小說長度可達 300 萬字以上。

技術棧：SwiftUI + iOS 17+，不使用 UIKit 除非必要。

---

## 使用者輪廓

- 台灣 25-40 歲，習慣長時間閱讀網路小說（玄幻、都市、仙俠類）
- 主要場景：睡前閱讀（暗房）、通勤聽書（背景播放）、午休翻幾頁
- 每天使用 1-3 小時，對閱讀舒適度極度敏感
- 痛點：現有閱讀 App 的 UI 太雜亂、廣告多、夜間模式刺眼、TTS 控制不方便

---

## 設計原則

1. **閱讀優先**：UI 必須在閱讀時幾乎消失，文字是唯一主角
2. **單手操作**：所有核心操作都要能用右手拇指在 6.7 吋螢幕上完成
3. **暗色系為主**：使用者有大量時間在暗房使用，亮色主題是次要的
4. **台灣審美**：避免中國 App 常見的紅金配色和繁複裝飾，偏向日系簡潔或台灣獨立書店的質感
5. **TTS 是一等公民**：語音控制不是藏在選單裡的附加功能，它的操作體驗要比肩音樂播放器

---

## 需要設計的畫面（共 5 個主要畫面）

### 畫面 1：書庫（LibraryView）

功能需求：
- 顯示所有已匯入的小說，每本顯示書名、閱讀進度、最後閱讀時間
- 匯入新書按鈕（觸發系統檔案選取器）
- 左滑刪除、長按重新命名
- 正在「聽」的書需要有明顯的視覺指示（例如聲波動畫）

設計方向：
- 參考「書架」而非「列表」的意象，但不要做成擬物化書架
- 封面區域可以用演算法根據書名生成抽象色塊或漸層，因為 txt 檔沒有封面圖
- 進度用細微的橫條或弧形表示，不要太搶眼
- 空狀態（沒有書時）要有引導匯入的設計

### 畫面 2：閱讀器（ReaderView）

功能需求：
- 全螢幕顯示小說文字，點擊螢幕中央切換工具列顯示/隱藏
- 支援兩種閱讀模式：上下捲動（Scroll）和左右翻頁（Page）
- 上方工具列：返回、書名、目錄按鈕
- 下方工具列：閱讀進度條、TTS 迷你控制列
- 當 TTS 朗讀時，當前段落需要有高亮標記

設計方向：
- 這是使用者停留最久的畫面，排版必須完美
- 文字區域要有足夠的邊距，行距要舒適
- 工具列出現/消失需要流暢的過場動畫
- 進度條要能即時拖動跳轉，並顯示章節資訊
- TTS 高亮不要太搶眼，用柔和的底色變化而非螢光色
- 翻頁模式需要有物理感的翻頁動畫

### 畫面 3：TTS 播放器（NarratorPlayerView）

功能需求：
- 從閱讀器底部上滑展開的完整 TTS 控制面板（類似 Apple Music 的 Now Playing）
- 播放/暫停、上一段/下一段
- 語速滑桿（0.8x ~ 2.0x）
- 語音選擇（曉臻/曉雨/雲哲 + 系統語音）
- TTS 引擎選擇（Edge TTS / 系統內建 / Azure）
- 睡眠計時器（15min / 30min / 45min / 1hr / 章節結束時）
- 顯示當前朗讀的段落文字預覽

設計方向：
- 這個面板的視覺語言要向音樂播放器致敬，但融入閱讀的氣質
- 背景可以有根據書名生成的模糊色彩（和書庫封面呼應）
- 控制按鈕要大，方便盲操作（通勤時不看螢幕也能按到）
- 收合狀態是一個迷你播放條，駐留在閱讀器底部

### 畫面 4：目錄側欄（ChapterListView）

功能需求：
- 從右側滑入的半透明側欄
- 顯示所有章節列表，當前章節有高亮
- 可搜尋章節
- 點擊跳轉到目標章節

設計方向：
- 300 萬字的小說可能有 1000+ 章，列表性能和快速捲動體驗很重要
- 需要字母索引或章節群組（卷/部分）
- 當前章節和已讀/未讀需要有視覺區別

### 畫面 5：設定頁（SettingsView）

功能需求：
- 字型大小滑桿（14pt ~ 32pt）
- 行距選擇（1.2x / 1.5x / 1.8x / 2.0x）
- 閱讀模式切換（捲動/翻頁）
- 背景主題切換：至少四種
  - 墨夜（純黑底淡灰字，OLED 省電）
  - 暖燈（深棕底暖白字，模擬檯燈下閱讀）
  - 紙白（米白底深灰字，日間閱讀）
  - 薄霧（深灰底淺灰字，低對比護眼）
- 字體選擇
- TTS 伺服器設定（Edge TTS server URL、API key 等）

設計方向：
- 設定面板最好是從閱讀器底部彈出的 sheet，而不是跳轉到新頁面
- 主題切換要有即時預覽效果
- 所有滑桿調整都要即時反映在背後的閱讀器上

## 設計系統需求

### 色彩
- 定義四套完整的閱讀主題色彩（背景、文字、強調色、UI 元素色）
- 主要品牌色用於書庫和非閱讀場景
- TTS 播放器使用的強調色要和閱讀主題聯動

### 字型
- UI 字型和閱讀字型要分開定義
- UI 用系統字型即可，但閱讀字型要支援切換
- 中文排版要處理好標點符號壓縮和行首行尾禁則

### 動畫
- 工具列出現/消失：fade + slide，duration 0.25s
- 翻頁動畫：curl effect 或 slide
- TTS 播放器展開/收合：interactive spring animation
- 書庫卡片的 onAppear：staggered fade in
- TTS 朗讀高亮：柔和的背景色漸變過渡

### 圖標
- 使用 SF Symbols，不引入第三方圖標
- 播放控制的圖標要比標準尺寸大 1.5 倍

---

## 互動細節

- 閱讀器支援長按選取文字
- 底部 TTS 迷你播放條支援左滑跳下一段、右滑跳上一段
- 書庫列表支援下拉刷新手勢（重新掃描匯入目錄）
- 閱讀器雙指捏合調整字型大小
- 搖動手機（shake gesture）快速切換播放/暫停

---

## 不要做的事

- 不要使用任何紅色、金色作為主色調（避免中國風）
- 不要使用 Tab Bar 導航，這個 App 只有兩層：書庫 → 閱讀器
- 不要在閱讀器放任何浮動按鈕（FAB）
- 不要使用漸層作為文字背景
- 不要使用圓角超過 16pt 的卡片
- 不要模仿微信讀書或起點讀書的設計語言
